# 🚀 Workspace 초고속 개발 편의성 가이드

이 워크스페이스는 복잡하고 거대한 코드베이스(회사 안드로이드/백엔드 프로젝트 등)에서 **내가 원하는 코드를 빛의 속도로 찾고, 반복적인 터미널 작업을 최소화하여 칼퇴를 돕기 위해** 만들어진 궁극의 개발 편의 툴킷입니다.

방대하게 얽혀있는 명령어들을 잊어버리세요. 내일부터는 이거 하나로 모든 게 끝납니다.

---

## ⚙️ 1. 초기 세팅 (새 서버에서 딱 한 번!)
저장소를 클론한 뒤, 필요한 부가 패키지를 설치합니다.

```bash
git clone [사용자님의 저장소 주소] workspace
cd workspace
npm install
```

## 🪄 2. 환경 장착 (마법의 명령어)
매번 터미널을 열 때마다 아래 명령어 한 줄만 치면 나만의 쾌적한 개발 환경이 즉시 장착됩니다.

```bash
source setup.sh
```

---

## ⚡ 3. 핵심 편의 기능 TOP 4 및 실전 활용 예시

`source setup.sh`를 실행하면 다음의 사기적인 단축 키워드들이 활성화됩니다. 아래 **[실전 예시]**처럼 바로 따라 쳐보세요!

### 🔍 (1) 초고속 맞춤형 코드 검색 (Grep 단축키)
기존에 `grep`을 쓰면 빌드 캐시(`.git`, `out`, `__pycache__` 등)까지 뒤지느라 너무 오래 걸렸죠? 불필요한 캐시/임시 폴더는 알아서 무시하고 원하는 파일 속 내용만 0.1초 만에 찾아냅니다.

*   `kgrep` : 코틀린(`.kt`) 파일 속에서만 검색
*   `jgrep` : 자바(`.java`) 파일 검색
*   `resgrep` : 안드로이드 리소스(`.xml`) 검색
*   `pygrep` : 파이썬(`.py`) 파일 검색
*   `cgrep` : C/C++(`.c`, `.cpp`, `.h`) 파일 검색
*   `ggrep` : 그래들 설정(`.gradle`) 파일 검색

> **💡 [실전 예시]**
> *상황: 자바 소스 코드 중에 "NullPointerException"을 처리하는 부분을 찾고 싶을 때*
> ```bash
> jgrep "NullPointerException"
> ```
> *상황: 안드로이드 xml 속성 중 "btn_submit"이라는 ID를 가진 리소스 위치를 찾을 때*
> ```bash
> resgrep "btn_submit"
> ```

### 🗂️ (2) 압축 풀기 스트레스 제로 (`extract`)
확장자(`.tar.gz`, `.zip`, `.rar`, `.7z`)마다 각기 다른 압축 해제 명령어를 구글링할 필요가 없습니다. 폴더까지 알아서 생성하고 풀어줍니다.

> **💡 [실전 예시]**
> *상황: 동료가 넘겨준 `server_logs_v2.tar.gz` 파일을 당장 풀어봐야 할 때*
> ```bash
> extract server_logs_v2.tar.gz
> ```
> *(결과: `server_logs_v2` 라는 폴더가 자동 생성되며 그 안에 깔끔하게 압축이 풀립니다)*

### 📂 (3) 프로젝트 순간이동 및 디렉토리 생성
폴더 이동에 낭비되는 스트레스를 줄여줍니다.

*   `workspace [프로젝트명]` : 복잡한 디렉토리를 찾아 들어갈 필요 없이 즉시 해당 프로젝트 폴더로 순간이동하며 터미널 탭 제목도 알아보기 쉽게 변경됩니다.
*   `mkdir_cd [폴더명]` : 폴더를 생성(`mkdir`)함과 동시에 바로 그 안으로 진입(`cd`)까지 원스텝으로 처리합니다.

> **💡 [실전 예시]**
> *상황: `InsightDeal` 프로젝트 폴더로 바로 넘어가고 싶을 때*
> ```bash
> workspace InsightDeal
> ```
> *상황: `test_module`이라는 폴더를 새로 파서 바로 그 안에서 작업하고 싶을 때*
> ```bash
> mkdir_cd test_module
> ```

### 📱 (4) 안드로이드 디바이스 제어 (옵션)
안드로이드 개발 시 숨막히게 귀찮은 테스트들을 명령어 한방에 처리합니다.

> **💡 [실전 예시]**
> *상황: 현재 물려있는 테스트 폰의 CPU 부하 시스템 스트레스 테스트를 발생시키고 싶을 때*
> ```bash
> test_cpu_stress.sh
> ```

## 🤖 4. (부가기능) Gemini AI 스마트 연동
단순 검색을 넘어 파일 구조 분석 후 변경점을 AI에게 자동으로 던지고 싶다면 활용하세요.

> **💡 [실전 예시]**
> *상황: 내가 방금 수정한 코드를 AI에게 자동 코드리뷰 시키고 싶을 때*
> 1. 백그라운드 쉘 탭을 하나 열고 `node bridge.cjs` 실행
> 2. 원래 코딩하던 탭에서, 코드 커밋 전에 아래 명령어 입력
> ```bash
> send_diff.sh
> ```
> *(결과: 내가 방금 타이핑한 코드 변경점만 AI로 발송되어 리뷰를 진행함)*
